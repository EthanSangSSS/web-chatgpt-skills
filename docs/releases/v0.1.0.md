# v0.1.0 — Public OSS readiness baseline

This release note is prepared for the first public baseline release of `web-chatgpt-skills`.

## Release scope

`v0.1.0` should be cut from a validated `main` commit after the follow-up readiness PR lands.

Included baseline surfaces:

- canonical `CATALOG.yaml` skill registry;
- self-contained `<skill>/SKILL.md` package layout;
- public governance files: `LICENSE`, `CONTRIBUTING.md`, `SECURITY.md`, `.github/CODEOWNERS`, and PR template;
- reviewer-facing OpenAI Codex for OSS evidence packet;
- maintainer runbook;
- roadmap;
- instruction-level security threat model;
- release checklist;
- prompt security review checklist;
- public-safe examples;
- catalog validator;
- public-safety scan;
- unsafe-claim regression fixtures;
- changed-skill reviewer aid.

## Validation before publishing

Run from the repository root:

```bash
python3 scripts/validate_catalog.py
python3 scripts/check_public_safety.py
python3 scripts/check_unsafe_claims.py
python3 scripts/list_changed_skills.py
```

Also check:

```bash
git diff --check
git status --short
```

A release should only be published if validation is green and the worktree is clean.

## Suggested GitHub release command

After syncing local `main` to the validated remote HEAD:

```bash
gh release create v0.1.0 \
  --repo EthanSangSSS/web-chatgpt-skills \
  --title "v0.1.0 — Public OSS readiness baseline" \
  --notes-file docs/releases/v0.1.0.md
```

## Honesty boundary

This release does not claim stars, downloads, production adoption, external users, or OpenAI endorsement. It is an early-stage public baseline for safer AI-agent skill packaging and maintainer workflow review.

## Known limitations

- The validation scripts are lightweight and dependency-free; they do not replace full security review.
- The public-safety scan is not a complete secret scanner.
- Human review is still required before using generated skills, handoffs, examples, or audit outputs in production contexts.
